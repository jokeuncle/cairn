Cairn Inspector

What the inspector is actually showing

The inspector is a visual debugger for Cairn's document index. It shows how raw repository documents become a structured graph, and why a retrieval result has evidence beyond a single vector hit.

Document section Extracted entity Cross-reference Query signal
Core idea Inspector is not the retrieval interface itself. It is the window into the index that lets maintainers verify whether Cairn understood a repository's documents correctly.

Parse structure

section tree + source paths
Tree

Preserves source order, section depth, anchors, and heading paths.

Vectors

Give semantic recall, but are only one signal in repo ranking.

Graph

Entities and cross-references reveal relationships between sections.

Inspector

Helps spot bad splitting, noisy entities, weak xrefs, and ranking drift.
